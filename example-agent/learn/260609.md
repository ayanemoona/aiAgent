# UV와 가상환경(.venv) 정리

## 1. 프로젝트 시작하기

프로젝트 폴더(example-agent)로 이동한 후 사용한다.

```bash
uv sync
```

### uv sync란?

`pyproject.toml` 파일에 적혀 있는 dependencies(의존성, 프로젝트 실행에 필요한 패키지)를 읽어서 자동으로 설치한다.

예시

```toml
dependencies = [
    "openai",
    "requests"
]
```

이라면 `uv sync`를 실행할 때 자동으로 설치된다.

---

## 2. 패키지 추가하기

```bash
uv add 패키지명
```

예시

```bash
uv add openai
uv add requests
```

### uv add란?

새로운 패키지를 설치하는 명령어이다.

설치와 동시에 `pyproject.toml`에도 자동으로 기록된다.

---

# .venv란?

`uv sync`를 처음 실행하면 `.venv` 폴더가 생성된다.

```text
example-agent
 ├─ .venv
 ├─ main.py
 ├─ pyproject.toml
 └─ uv.lock
```

이 `.venv`가 바로 가상환경(Virtual Environment)이다.

---

## 가상환경을 사용하는 이유

프로젝트마다 필요한 패키지 버전이 다를 수 있다.

예를 들어

프로젝트 A

```text
openai 1.0
```

프로젝트 B

```text
openai 2.0
```

를 사용해야 할 수도 있다.

가상환경을 사용하면 프로젝트별로 독립적인 패키지 공간을 만들 수 있다.

따라서 서로 충돌하지 않는다.

---

## 코드 실행 시

프로그램을 실행할 때는

```text
.venv 안에 설치된 Python
+
.venv 안에 설치된 패키지
```

를 사용하여 실행한다.

---

# uv.lock이란?

`uv.lock` 파일은 프로젝트 실행에 필요한 패키지들의 정확한 버전을 기록하는 파일이다.

예시

```text
openai 1.93.0
requests 2.32.4
```

팀원이 같은 프로젝트를 받을 경우

```bash
uv sync
```

만 실행하면 동일한 버전으로 설치된다.

즉,

```text
프로젝트 실행 환경을 고정하는 파일
```

이라고 이해하면 된다.

# VS Code에서 가상환경 사용하기

## 방법 1 (권장)

### Python 인터프리터 선택

```text
Ctrl + Shift + P
```

↓

```text
Python: Select Interpreter
```

↓

가상환경 선택

```text
example-agent (Python 3.13.11)
.venv\Scripts\python.exe
```

선택

---

### 선택 확인

터미널에서 실행

```bash
.\.venv\Scripts\python.exe --version
```

결과

```text
Python 3.13.11
```

---

# 방법 2

PowerShell에서 가상환경 직접 활성화

```powershell
.\.venv\Scripts\Activate.ps1
```

활성화되면

```powershell
(.venv) PS C:\example-agent>
```

처럼 표시된다.

---

# 방법 3

uv가 자동으로 가상환경 사용

```bash
uv run main.py
```

실행 시

```text
자동으로 .venv 사용
```

따라서 별도 활성화 없이도 실행 가능하다.

# Jupyter Notebook에서 가상환경 사용하기

Jupyter는 Python 인터프리터와 별도로 Kernel(커널)을 사용한다.

따라서 가상환경을 선택해줘야 한다.

---

## 1. ipykernel 설치

```bash
uv add ipykernel --dev
```

### --dev란?

프로젝트 실행에는 필요 없지만 개발자가 사용하는 패키지를 의미한다.

예시

```text
ipykernel
pytest
black
```

---

## 2. Kernel 선택

Jupyter Notebook(.ipynb) 파일 열기

↓

오른쪽 위

```text
Select Kernel
```

클릭

↓

```text
example-agent (Python 3.13.11)
```

선택

---

## 왜 Kernel을 선택해야 할까?

Kernel이 다르면

```text
VS Code는 .venv 사용 중
```

인데

```text
Jupyter는 다른 Python 사용 중
```

일 수 있다.

이 경우 설치한 패키지를 찾지 못하는 문제가 발생한다.

따라서 Jupyter에서도 반드시 프로젝트의 가상환경(.venv)을 선택해야 한다.
