2. 실제로 함수 실행 
arguments는 문자열로 들어옴 ex) "{'city' : 'Spain' }
실제로 필요 한 건 객체 -> import json
json string을 load
json.loads()를 하는 이유가 "파이썬이 문자열을 못 받아서"가 아니라 "JSON 문자열을 dict로 바꿔서 **arguments로 함수를 호출하기 위해서
try:
            arguments = json.loads(arguments)
        except json.JSONDecodeError:
            arguments = {} 

받은 문자열을 실제 python 객체로 바꿈
get_weather(city="Seoul") = get_weather(**arguments) 으로 바뀜 
자바와 달리 파이썬은 동적 타입 언어라 함수 호출 시 매개변수 타입을 엄격하게 강제하지 않는다. 따라서 JSON 문자열을 json.loads()로 Python 객체(dict)로 변환한 뒤 **arguments 형태로 함수에 전달하면 된다.
**arguments 형태인 이유는 함수는 python 딕셔너리를 받지 않음 그냥 문자열 하나만 받으면 됨 seoul 하나 
** 표기는 딕셔너리를 키워드 인자로 바꿔주기 때문

Content Part 형식
최신 API는 이런 형태를 지원합니다.
{
    "role": "user",
    "content": [
        {
            "type": "text",
            "text": "안녕"
        }
    ]
}

여기서는 반드시
"type": "text"
가 있어야 합니다.
json 형태에 type이 없어 바로 에러 발생
